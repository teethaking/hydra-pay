# Contributing Guidelines

To keep the Hydra-Pay codebase clean and maintainable, please follow these steps:

1. **Branching:** Use `feat/` for new features and `fix/` for bugs.
2. **Pull Requests:** Provide a clear description of the "Why" behind your changes.
3. **Coding Style:** We use Prettier for TypeScript and Solhint for Solidity.
4. **Security:** Never commit private keys or sensitive environment variables.
